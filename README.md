# NorthStar Urban Mobility and Logistics Analytics

## Project Overview

This project investigates operational inefficiencies within NorthStar Urban Mobility and Logistics using integrated relational analytics, Python-based operational intelligence, and MongoDB Atlas NoSQL redesign.

The analysis focuses on identifying root causes behind delays, customer complaints, failed deliveries, operational bottlenecks, fragmented reporting systems, and inconsistent operational visibility across structured and semi-structured data environments.

---

## Technologies Used

- SQL within R
- R analytics and visualisation
- Python (Pandas, NumPy, Matplotlib, Plotly)
- MongoDB Atlas
- PyMongo
- Google Colab
- GitHub

## Analytical Workflow

1. Business problem identification
2. Data understanding and preprocessing
3. SQL within R analytics
4. R statistical analysis and visualisation
5. Python integrated operational analytics
6. MongoDB Atlas NoSQL implementation
7. Query optimisation and performance analysis
8. Strategic recommendations and operational redesign

---

## Project Objective

The objective of this project is to redesign NorthStar’s fragmented operational intelligence environment by integrating relational analytics and NoSQL operational modelling to support scalable analysis, improved visibility, and management-level decision-making.
